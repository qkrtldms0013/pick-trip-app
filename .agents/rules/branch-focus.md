# Branch Focus Rule

현재 브랜치가 `feat/<issue_number>` 형식일 때, 해당 브랜치의 기능 범위를 벗어나는 작업 요청은 수락하지 않는다.

## 규칙

- 현재 브랜치명의 이슈 번호로 대상 이슈를 찾고, 그 이슈 내용으로 작업 범위를 판단한다.
  - 예: `feat/12` → 허용 범위: 이슈 #12에 정의된 작업
- 사용자가 현재 브랜치 기능과 **무관한 기능 개발**을 요청하면:
  1. 요청이 현재 브랜치 범위를 벗어남을 명확히 경고한다.
  2. 작업을 진행하지 않는다.
  3. 새 브랜치를 생성하여 작업할 것을 안내한다.

## 경고 메시지 예시

```
⚠️ 브랜치 포커스 경고

현재 브랜치: feat/12
요청하신 작업은 현재 브랜치의 범위(이슈 #12)를 벗어납니다.

새 이슈를 생성한 후 해당 번호로 브랜치를 만들어 작업해 주세요:
  git checkout -b feat/<issue_number>
```

## 예외

- 현재 브랜치가 `main`이거나 `feat/` 접두사가 없는 경우 이 규칙을 적용하지 않는다.
- 버그 수정(`fix/`), 리팩토링(`refactor/`) 등 다른 접두사 브랜치도 동일한 원칙을 따른다.
